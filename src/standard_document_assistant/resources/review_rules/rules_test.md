# 全文

1. 按照文件要素要求，全文按顺序包含以下内容： 封面、目次、前言、引言、正文、结尾部分。其中，前言、正文是必填项，目次、引言、结尾部分是可选项。

2. 全文不能够出现错别字、错误标点。文件中使用的汉字应为规范汉字。

3. 全文要遵循前后对应关系，确保前后文一致性。

# 封面

1. 在封面中应标明以下必备信息：文件名称、文件的层次或类别（如“中华人民共和国国家标准”、“中华人民共和国xxxx行业标准”、“中华人民共和国国家标准化指导性技术文件”、“xxxx地方标准”等字样）、文件代号（如“XX”、“XX/T”、“XX/Z”等，X表示大写拉丁字母）、文件编号、国际标准分类(ICS)号、中国标准文献分类(CCS)号、发布日期、实施日期、发布机构等。

2. 如果文件代替了一个或多个文件，在封面中应标明被代替文件的编号。

3. 文件编号由文件代号、顺序号及发布年份号构成。文件代号由大写拉丁字母和/或符号“/”组成，顺序号由阿拉伯数字组成（可能含有小数点），发布年份号由四位阿拉伯数字组成，顺序号和年份号之间使用一字线形式（——或-）的连接号。例如：GB/TXXXXXX-XXXXX。

4. 国家标准、行业标准的封面还应标明文件名称的英文译名。

# 目次

1. 目次中的内容（章节号及标题）应该与正文中的内容保持一致。不能在目次中列出正文中没有的内容，也不能在正文中列出目次中没有的内容。

2. 根据所形成的文件的【具体全体正文内容情况】，应依次对下列内容建立目次列表【并不是要求必须包含下列所有内容，主要强调排列顺序】：
   a）前言， 
     b）引言， 
     c）章编号和标题， 
     d）条编号和标题，
     e）附录编号、“（规范性）”/“（资料性）”和标题， 
     f）附录条编号和标题， 
     g）参考文献， 
     h）索引， 
     i）图编号和图题(含附录中的)，
     j）表编号和表题（含附录中的）。

3. 在目次中不应列出正文“术语和定义”章节下的条目编号和术语。

# 前言

1. 【必须含有】文件起草所依据的标准。具体表述为（请忽略标点符号格式）：“本文件按照GB/T 1.1—2020《标准化工作导则第1部分：标准化文件的结构和起草规则》的规定起草。”

2. 【选择必须】文件与其他文件的关系。需要说明以下两方面的内容：
   - 与其他标准的关系； 
   - 分为部分的文件的每个部分说明其所属的部分并列出所有已经发布的部分的名称。

3. 【选择必须】文件与代替文件的关系。需要说明以下两方面的内容：
   - 给出被代替、废止的所有文件的编号和名称； 
   - 列出与前一版本相比的主要技术变化。

4. 【必须含有】有关专利的说明。具体表述为（请忽略标点符号格式）：“请注意本文件的某些内容可能涉及专利。本文件的发布机构不承担识别专利的责任。”

5. 【必须含有】文件的**提出信息(可省略)和归口信息**。对于由全国专业标准化技术委员会提出或归口的文件，应在相应技术委员会名称之后给出其国内代号，使用下列适当的表述形式（请忽略括号格式）：
   - “本文件由全国 $xx$ 标准化技术委员会(SAC/TC XXX)提出。” 
   - “本文件由 $xx$ 提出。” 
   - “本文件由全国 $xx$ 标准化技术委员会(SAC/TC XXX)归口。” 
   - “本文件由 $xx$ 归口。”

6. 【必须含有】文件的**起草单位**和**主要起草人**，使用下列表述形式：
   - “本文件起草单位：……。” 且起草单位不得少于3个。
   - “本文件主要起草人：……。” 且主要起草人不超过15人。

7. 【必须含有】文件及其所代替或废止的文件的历次版本发布情况，使用下列表述形式：
   - “——xxxx年首次发布为......”
   - “——xxxx年......”
   - “——本次为第x次修订......”

# 引言

1. 【注意】引言内容为可选项。但是分为部分的文件的每个部分应设置引言，即当前文件的中文名称中若出现“第X部分”则【必须】有引言部分。

2. 不应包含要求型条款，即不应出现“不应”、“不准许”、“不应该”、“应”、“应该”、“准许”这类能愿动词。

3. 引言不应给出章编号。当引言的内容需要分条时，应仅对条编号，编为0.1、0.2等。

4. 若含有引言部分，则需要在引言中说明编制该文件的原因、编制目的、分为部分的原因以及各部分之间关系等事项的说明；或含有文件技术内容的特殊信息或说明。

# 正文

## 范围

1. 范围需要作为第1章内容。范围的陈述应简洁，以便能作为内容提要使用。

2. 范围应表述为一系列事实的陈述，使用陈述型条款，不应包含要求、指示、推荐和允许型条款，即只能出现“能”、“不能”、“可能”、“不可能”这类能愿动词表述和陈述句。

3. 范围的陈述应使用如下列所示的适当的表述形式：
   - “本文件规定了……的要求/特性/尺寸/指示”；
   - “本文件确立了……的程序/体系/系统/总体原则”；
   - “本文件描述了……的方法/路径”； 
   - “本文件提供了……的指导/指南/建议”； 
   - “本文件给出了……的信息/说明”； 
   - “本文件界定了……的术语/符号/界限”。 
4. 其中文件适用界限的陈述应使用下列适当的表述形式：
   - “本文件适用于……”； 
   - “本文件不适用于……”。

## 规范性引用文件

1. 规范性引用文件需要作为第2章内容。

2. 【必须含有】规范性引用文件清单应由以下引导语引出（忽略标点符号格式）：
   - “下列文件中的内容通过文中的规范性引用而构成本文件必不可少的条款。其中，注日期的引用文件，仅该日期对应的版本适用于本文件；不注日期的引用文件，其最新版本（包括所有的修改单）适用于本文件。” 
   - 如果不存在规范性引用文件，应在章标题下给出以下说明： “本文件没有规范性引用文件。”

3. 文件清单中应列出该文件中规范性引用的每个文件，列出的文件之前不给出序号。下列表述形式属于规范性引用:
   a）任何文件中，由要求型或指示型条款（即使用“不应”、“不准许”、“不应该”、“应”、“应该”、“准许”或祈使句表述）提及文件;
     b）规范标准中，由“按”或“按照”提及试验方法类文件;
     示例:“甲醛含量按GB/T 2912.1—2009描述的方法测定应不大于20mg/kg”,其中的GB/T 2912.1—2009为规范性引用的文件。
     c）指南标准中，由推荐型条款（即使用“宜”、“不宜”、“推荐”、“不推荐”、“建议”、“不建议”这类表述）提及文件;
     d）任何文件中，在要素“术语和定义”中由引导语提及文件。
     文件中所有规范性引用的文件，无论是注日期，还是不注日期，均应在要素“规范性引用文件”中列出

4. 根据文件中引用文件的具体情况，文件清单中应选择列出下列相应的内容：
   - 注日期的引用文件，给出“文件代号、顺序号及发布年份号和/或月份号”以及“文件名称”； 
   - 不注日期的引用文件，给出“文件代号、顺序号”以及“文件名称”； 
   - 不注日期引用文件的所有部分，给出“文件代号、顺序号”和“（所有部分）”以及文件名称中的“引导元素(如果有)和主体元素”； 
   - 引用国际文件、国外其他出版物，给出“文件编号”或“文件代号、顺序号”以及“原文名称的中文译名”，并在其后的圆括号中给出原文名称。

5. 根据文件中引用文件的具体情况，文件清单中列出的引用文件的排列顺序为【并不是要求必须包含下列所有内容，主要强调排列顺序】：
   a）国家标准化文件， 
     b）行业标准化文件， 
     c）本行政区域的地方标准化文件（仅适用于地方标准化文件的起草）， 
     d）团体标准化文件, 
     e）ISO、ISO/IEC或IEC标准化文件， 
     f）其他机构或组织的标准化文件， 
     g）其他文献。 
     其中，国家标准、ISO或IEC标准按文件顺序号排列；行业标准、地方标准、团体标准、其他国际标准化文件先按文件代号的拉丁字母和/或阿拉伯数字的顺序排列，再按文件顺序号排列。

6. 起草文件时不应引用已被代替或废止的文件。起草文件时不应规范性引用法律,行政法规、规章和其他政策性文件,也不应普遍性要求符合法规或政策性文件的条款。诸如“……应符合国家有关法律法规”的表述是不正确的。

## 术语和定义

1. 术语和定义需要作为第3章内容。

2. 【必须含有】根据列出的术语和定义以及引用其他文件的具体情况，术语条目应分别由下列适当的引导语引出：
   - “下列术语和定义适用于本文件。”（如果仅该要素界定的术语和定义适用时） 
   - “……界定的术语和定义适用于本文件。”（如果仅其他文件中界定的术语和定义适用时） 
   - “……界定的以及下列术语和定义适用于本文件。”（如果其他文件以及该要素界定的术语和定义适用时） 
   - 如果没有需要界定的术语和定义，应在章标题下给出以下说明： “本文件没有需要界定的术语和定义。”

3. 术语条目宜按照概念层级分类和编排，如果无法或无须分类可按术语的汉语拼音字母顺序编排。术语条目的排列顺序由术语的条目编号来明确。条目编号应在章或条编号之后使用下脚点加阿拉伯数字的形式。注：术语的条目编号不是条编号。

4. 每个术语条目应包括四项内容：条目编号、术语、英文对应词、定义，根据需要还可增加其他内容。按照包含的具体内容术语条目中应依次给出【并不是要求必须包含下列所有内容，主要强调排列顺序】：
   a）条目编号， 
     b）术语， 
     c）英文对应词， 
     d）符号， 
     e）术语的定义， 
     f）概念的其他表述形式（如图、数学公式等）， 
     g）示例， 
     h）注， 
     i）来源等。

5. 术语条目不应编排成表的形式，它的任何内容均不准许插入脚注。

6. 列出的术语在完整文件中至少使用两次。

7. 定义应使用陈述型条款，既不应包含要求型条款，也不应写成要求的形式。附加信息应以示例或注的表述形式给出。定义中不能重复术语本身，定义开头不能出现“是”、“是指”字样。

## 符号和缩略语

1. 【注意】符号和缩略语为可选部分，这一要素用来给出为理解文件所必需的、文件中使用的符号和缩略语的说明或定义，由引导语和带有说明的符号和/或缩略语清单构成。如果需要设置符号或缩略语，宜作为文件的第4章。如果为了反映技术准则，符号需要以特定次序列出，那么该要素可以细分为条，每条应给出条标题。根据编写的需要，该要素可并入第3章“术语和定义”。

2. 根据列出的符号、缩略语的具体情况，符号和/或缩略语清单应分别由下列适当的引导语引出：
   - “下列符号适用于本文件。”（如果该要素列出的符号适用时） 
   - “下列缩略语适用于本文件。”（如果该要素列出的缩略语适用时） 
   - “下列符号和缩略语适用于本文件。”（如果该要素列出的符号和缩略语适用时）

3. 无论该要素是否分条，清单中的符号和缩略语之前均不给出序号，且宜按下列规则以字母顺序列出：
   a）大写拉丁字母置于小写拉丁字母之前 $(A,a,B,b)$ 等； 
     b）无角标的字母置于有角标的字母之前，有字母角标的字母置于有数字角标的字母之前（ $B, b, C, C_{\mathrm{m}}, C_{2}, c, d, d_{\mathrm{ext}}, d_{\mathrm{int}}, d_{1}$ 等）； 
     c）希腊字母置于拉丁字母之后 $(Z,z,A,\alpha ,B,\beta ,\dots ,\Lambda ,\lambda$ 等）； 
     d）其他特殊符号置于最后。

4. 符号和缩略语的说明或定义宜使用陈述型条款，不应包含要求和推荐型条款。


# 格式规范

1. 章是文件层次划分的基本单元。应使用从“1”开始的阿拉伯数字对章编号。章编号应从“范围”一章开始，一直连续到附录之前。每一章均应有章标题,并应置于编号之后。

2. 条是章内有编号的细分层次。条可以进一步细分,细分层次不宜过多,最多可分到第五层次。条编号应使用阿拉伯数字并用下脚点与章编号或上一层次的条编号相隔。同一层次的条有无标题应统一。**注：在“术语和定义”章节中，条目编号不是条编号。**

3. 段是章或条内没有编号的细分层次。为了不在引用时产生混淆,不宜在章标题与条之间或条标题与下一层次条之间设段(称为“悬置段”)。**注:“术语和定义”“符号和缩略语"中的引导语不是悬置段。**

4. 列项是段中的子层次,用于强调细分的并列各项中的内容。列项应由引语和被引出的并列的各项组成。具体形式有以下两种（列项的最后一项均由句号结束）：

   - 后跟句号的完整句子引出后跟句号的各项；
   - 后跟冒号的文字引出后跟分号或逗号的各项。

   列项可以进一步细分为分项，这种细分不宜超过两个层次。列项中各项的内容应并列、简短，同一标准的列项格式应统一。在列项的各项之前应标明列项符号或列项编号。列项符号为破折号(——)或间隔号(·);列项编号为字母编号[即后带半圆括号的小写拉丁字母，如a）、b）等]或数字编号[即后带半圆括号的阿拉伯数字，如1）、2）等]。示例：

   ```
   # 示例1:
   导向要素中图形符号与箭头的位置关系需要符合下列规则。
   a） 当导向信息元素横向排列，并且箭头指:
     1） 左向(含左上、左下)，图形符号应位于右侧;
     2） 右向(含右上、右下),图形符号应位于左侧;
     3） 上向或下向，图形符号宜位于右侧。
   b） 当导向信息元素纵向排列，并且箭头指:
     1）下向(含左下、右下)，图形符号应位于上方;
     2） 其他方向，图形符号宜位于下方。
   
   # 示例2:
   下列仪器不需要开关:
   ——正常操作条件下，功耗不超过10W的仪器;
   ——任何故障条件下使用2min,测得功耗不超过50W的仪器;
   ——连续运转的仪器。
   ```

5. 目次中列出的各项内容标注的页码与正文中相应内容的页码需要对应。
