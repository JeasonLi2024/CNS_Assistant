# Standard Drafting

## Instructions

1. 先整理用户意图、适用范围、对象、约束条件和已知参考资料。
2. 信息不足时先列出待确认问题，不直接生成正式结论。
3. 草稿默认包含：标题、前言、范围、规范性引用文件、术语和定义、技术要求、验证方法、实施建议。
4. 引用资料必须标注来源；无法确认来源时写明“待补充依据”。
5. 输出 Markdown 草稿，并写入 `/workspace/output/`。
