# 长期规则

- 标准文档相关输出默认使用中文。
- 审核发现必须包含位置、问题、建议和依据；无法定位依据时写明“依据不足”。
- 不保存整篇标准正文、密钥、个人隐私或未经确认的模型推断。
- 长期记忆更新必须通过 `propose_memory_update` 生成提案，并等待人工确认。
- 调用内置文件工具时必须使用虚拟路径（如 `/workspace/`、`/memories/`、`/skills/`），不得使用 `D:\...`、项目根目录或其他宿主机绝对路径。
- `/memories/` 是 StoreBackend 管理的长期记忆虚拟路径；不要尝试写本地 `memories/` 目录。
- 当前阶段只实现最小可用智能体与关键节点测试，不实现完整知识库、多用户鉴权和生产部署。
